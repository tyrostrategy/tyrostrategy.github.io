# i18n Rules

## Absolute Rule
**Every user-visible string MUST use `t("key")` from react-i18next. Zero hardcoded Turkish or English text in components.**

This includes:
- Button labels, form labels, placeholders
- Toast messages, validation messages
- `aria-label`, `title`, `placeholder` attributes
- Tooltip text, tab labels, column headers
- Even email examples like `placeholder="ornek@tiryaki.com.tr"` must use `t()`

---

## Text Casing Convention — Sentence Case

**All UI text uses Sentence case** (ilk harf buyuk, geri kalan kucuk) unless it's a proper noun or brand name.

### Turkish (TR) Rules
| Element | Convention | Example |
|---------|-----------|---------|
| Button labels | Sentence case (single word = capitalized) | "Kaydet", "İptal", "Oluştur" |
| Page titles | Sentence case | "Proje yönetimi", "Aksiyon listesi" |
| Form labels | Sentence case | "Proje adı", "Başlangıç tarihi" |
| Placeholders | Sentence case + consistent verb | "Proje adını giriniz..." |
| Toast messages | Sentence case, always start with capital | "Proje başarıyla oluşturuldu" |
| Table headers | Sentence case | "Proje adı", "İlerleme", "Durum" |
| Status labels | Sentence case | "Yolunda", "Risk altında", "Gecikmeli" |
| Menu items | Sentence case | "Gelişmiş filtre", "Dışa aktar" |

### English (EN) Rules
| Element | Convention | Example |
|---------|-----------|---------|
| Button labels | Sentence case | "Save", "Cancel", "Create" |
| Page titles | Sentence case | "Project management", "Action list" |
| Form labels | Sentence case | "Project name", "Start date" |
| Placeholders | Sentence case | "Enter project name..." |
| Toast messages | Sentence case | "Project created successfully" |

### BANNED Patterns
- **ALL CAPS for buttons:** ~~"DIŞA AKTAR"~~ → "Dışa aktar"
- **Title Case for sentences:** ~~"Her Kelime Büyük"~~ → "Her kelime büyük"
- **Inconsistent toast casing:** all toasts start with capital letter
- **Mixed verb forms:** standardize to formal "giriniz"/"seçiniz" for placeholders

### Proper Nouns (always capitalized)
- Brand: TYRO, TTECH, Tiryaki
- Product: Dataverse, Supabase, Azure
- App sections used as proper nouns: Kokpit, Sihirbaz (when referring to the specific feature)

---

## Placeholder Verb Convention (TR)

Use consistent verb endings in Turkish placeholders:
| Action | Verb Ending | Example |
|--------|-------------|---------|
| Text input | "giriniz..." | "Proje adını giriniz..." |
| Selection | "seçiniz..." | "Durum seçiniz..." |
| Search | "arayınız..." | "İsim ile arayınız..." |

**Never mix** "girin" (command) with "giriniz" (formal). Always use formal.

## Workflow for Adding New Text

1. Design the key: `namespace.context.label` format (e.g., `kokpit.filter.sourceTurkiye`)
2. Add to BOTH `src/locales/tr.json` AND `src/locales/en.json` simultaneously
3. Use `t("key")` in the component — never string literals
4. Run `npm run i18n:check` to verify key sync (0 mismatches required)

## Key Naming Convention
```
common.*           → Shared labels (save, cancel, delete, loading, etc.)
nav.*              → Navigation/sidebar labels
kokpit.*           → Dashboard/cockpit page
kokpit.filter.*    → Advanced filter panel
projeler.*         → Project management
aksiyonlar.*       → Action management
wizard.*           → Report wizard / multi-step flows
auth.*             → Login/logout/auth messages
table.*            → Table headers, pagination
toast.*            → Toast notification messages
forms.*            → Form labels and placeholders
validation.*       → Validation error messages
dates.*            → Date-related labels
settings.*         → Settings page
```

## Translation Value Rules
- **No duplicate semantic values** with different wording (e.g., "geciken" vs "gecikmiş" for "late" — pick one)
- **Namespace consistency:** button labels always in `common.*`, never in page-specific namespaces
- **Fallback default:** never use `t("key", "Fallback text")` pattern — if the key exists, it should work

---

## Tools Available
- `npm run i18n:check` — validates TR/EN key sync, exits 1 on mismatch
- `npm run i18n:scan` — scans TSX files for hardcoded Turkish characters (ç,ş,ğ,ı,ö,ü)
- Dev mode: missing keys auto-warn to console via `[i18n] Missing key:` handler

## Config
- Locale files: `src/locales/tr.json`, `src/locales/en.json`
- i18n setup: `src/lib/i18n.ts`
- Fallback language: Turkish (tr)
- Type safety: `src/types/i18next.d.ts` provides t() autocomplete from tr.json
- User preference stored in `localStorage("tyro-locale")`
